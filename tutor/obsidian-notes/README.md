# Obsidian Knowledge Base

This directory contains automatically generated technical notes that help users understand the TutorClaw platform development process and key technical concepts.

## Purpose
- Provide accessible technical documentation for users
- Explain complex implementation details in user-friendly terms
- Support cross-disciplinary understanding (STEM and humanities)
- Enable users to learn from and build upon the platform

## Structure
- `Technical-Concepts/` - Explanations of key technical concepts
- `Architecture-Decisions/` - Rationale behind architectural choices  
- `Feature-Guides/` - User guides for specific features
- `Best-Practices/` - Recommended usage patterns
- `Troubleshooting/` - Common issues and solutions

## Automatic Synchronization
All notes are automatically generated from development logs in `/home/node/workspace/clawd/development-logs/` and synchronized with appropriate Obsidian linking, tagging, and formatting for optimal knowledge discovery.

## Cross-Disciplinary Focus
Notes are written to be accessible to both technical (STEM) and non-technical (humanities) users, with clear explanations, examples, and context provided for all technical concepts.
