# TUTOR 功能规格说明书

## 项目概述

**TUTOR** (Thinking Understanding Testing Optimizing Refining) 是一个面向科研人员的 AI 辅助研究工作流自动化工具，支持从 idea 生成、实验验证、论文审核到最终撰写的全流程。

**目标用户**：
- 文科研究者：文献综述、研究方向建议、论文写作辅助
- 理科研究者：代码复现、实验自动化、结果分析
- 跨学科研究者：整合多领域知识的综合研究辅助

**核心特性**：
- 四大模块化工作流，可独立或串联使用
- 本地优先设计，用户数据不上传云端
- 支持 OpenAI API 兼容的多种模型接入
- CLI + Web 双端界面（CLI为MVP优先）

---

## 工作流1：Idea 生成与评估 (Thinking)

### 功能描述
根据用户输入的研究方向或参考文献，自动生成多个候选研究 idea，并通过多模型辩论和评估进行排序。

### 输入
- 研究方向描述（文本）
- 参考论文（PDF文件或arXiv URL）
- 关键词列表（可选）

### 处理流程
```
1. 文献理解
   ↓ 解析参考论文，提取核心贡献和方法
2. 网络查询（可选，MVP暂不实现）
   ↓ 调用Tavily/Agent-Reach补充最新动态
3. 第一轮 Debate（模型A vs 模型B）
   ↓ 围绕研究方向展开讨论
4. 深度思考（模型C整合）
   ↓ 系统性分析讨论结果
5. 第二轮 Debate
   ↓ 针对初步idea进行可行性辩论
6. Idea 池生成
   ↓ 输出多个候选idea（5-10个）
7. 导师审批（模型D评估）
   ↓ 多维度评分（创新性、可行性、insight等）
8. 综合排序
   ↓ 按总分降序输出
```

### 输出
- 排序后的研究 idea 列表（JSON）
- 每个 idea 包含：
  - 标题
  - 简要描述
  - 创新点说明
  - 评分（创新性、可行性、insight）
  - 评估理由
- 完整辩论过程记录（可选）

### MVP 限制
- 仅使用本地模型调用，无网络查询
- Debate 轮次固定为2轮
- 评估维度固定为3个：创新性、可行性、insight

## 工作流2：自动化实验 (Testing)

### 功能描述
根据选定的 idea，自动复现相关代码、配置实验环境、执行实验并生成结果报告。

### 输入
- 选定的 idea ID
- 实验配置（超参数、对比方法、数据路径等）
- 可选：远程服务器SSH配置（MVP暂不支持）

### 处理流程
```
1. 环境检测
   ↓ 检查本地 Python 环境、GPU 可用性
2. 代码获取
   ↓ 根据 idea 关联的 GitHub 仓库或 arXiv 代码链接
3. 依赖安装
   ↓ 读取 requirements.txt 或 setup.py
4. 实验执行
   ↓ 运行训练/测试脚本，记录日志和指标
5. 结果分析
   ↓ 生成可视化图表（loss曲线、性能对比等）
6. 对比评估
   ↓ 与 baseline 方法对比，生成报告
```

### 输出
- 实验日志文件
- 结果图表（PNG/PDF）
- 指标汇总表（CSV）
- 对比评估报告（Markdown）

### MVP 限制
- 仅支持本地实验执行
- 不支持远程SSH部署
- 依赖安装失败时需用户手动介入
- 可视化图表仅支持Matplotlib基础图表

## 工作流3：论文审核校验 (Optimizing)

### 功能描述
对完成的实验进行多维度审核，评估论文的创新点、工程质量和学术表达，提供反馈和改进建议。

### 输入
- 实验ID或论文草稿文件
- 可选：审核重点（针对特定维度）

### 处理流程
```
1. 理论导师审核
   ↓ 评估创新点和理论贡献
2. 工程导师审核
   ↓ 评估代码质量和实验完整性
3. 学术导师审核
   ↓ 评估写作逻辑和文献相关性
4. 综合反馈汇总
   ↓ 生成结构化反馈报告
5. 返修迭代（可选）
   ↓ 根据反馈进行多轮修改
6. 主题确定
   ↓ 明确论文核心贡献和发表方向
```

### 输出
- 三个角色的独立审核报告
- 综合反馈摘要
- 需要改进的点清单（带优先级）
- 论文主题建议

### MVP 限制
- 单角色审核（由单一模型完成，不区分角色）
- 不支持多轮自动返修（需用户手动修改）
- 审核维度固定：创新性、方法严谨性、实验完整性、写作质量

## 工作流4：论文撰写 (Refining)

### 功能描述
根据确定的论文主题和实验结果，自动生成符合学术规范的论文初稿，并支持多轮审核和修改。

### 输入
- 论文主题和核心贡献描述
- 实验结果和图表
- 目标格式（Markdown/LaTeX，MVP仅支持Markdown）
- 可选：论文模板（IMRaD或自定义）

### 处理流程
```
1. 大纲生成
   ↓ 基于主题生成论文结构
2. 初稿撰写
   ↓ 按章节自动生成内容（引用、公式、图表说明）
3. 格式检查
   ↓ LaTeX/Word格式规范检查（MVP跳过）
4. 专家审核
   ↓ 各章节由专家角色评审
5. 语言润色
   ↓ 语法、用词、表达优化
6. 查重检测（可选）
   ↓ Turnitin/iThenticate格式报告（MVP跳过）
7. 用户审批
   ↓ 用户确认后定稿
```

### 输出
- 完整论文（Markdown格式，MVP）
- 审核意见记录
- 最终 PDF（通过pandoc转换，MVP可选）
- 参考文献列表（BibTeX）

### MVP 限制
- 仅支持 Markdown 输出
- 不包含 LaTeX 格式检查
- 不支持自动查重
- 专家审核仅作为可选步骤

## 非功能性需求

### 性能要求
- 工作流1单次执行 < 5 分钟（不含网络查询）
- 工作流2实验超时可配置，默认24小时
- 工作流3审核响应 < 30 秒
- CLI命令响应时间 < 2 秒

### 可扩展性
- 支持自定义模型接入（任何OpenAI兼容接口）
- 支持自定义工作流编排（通过配置文件）
- 插件机制预留（MVP阶段不实现）

### 安全性
- API密钥仅本地存储，不上传云端
- SSH凭证加密存储（MVP不涉及远程）
- 用户数据完全本地化，隐私保护

### 兼容性
- CLI：Linux/macOS/Windows (PowerShell)
- Python 3.9+
- 无需Docker等容器化依赖

## 验收标准

### 功能验收
- [ ] 工作流1：输入研究方向→输出5个以上带评分的idea
- [ ] 工作流2：能读取本地实验结果，生成分析报告
- [ ] 工作流3：能对论文草稿提供至少3个维度的反馈
- [ ] 工作流4：能生成完整Markdown论文初稿
- [ ] CLI所有命令能正常执行且输出符合预期

### 质量验收
- [ ] 单元测试覆盖率 > 70%
- [ ] 核心流程无明显卡顿
- [ ] 错误处理完善，异常信息清晰
- [ ] 配置文件支持热重载
